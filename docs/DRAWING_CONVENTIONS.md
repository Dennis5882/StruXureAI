## 1. 레이어 명명 관례

부재 분류는 `src/utils/geometry.ts`의 `classifyLayer()` 하나로 통일한다(자동필터·추출이 같은 함수 사용 — 불일치 방지).

### 1-1. 부재 키워드 (한/영/중·번체)
| 부재 | 키워드 |
|---|---|
| 기둥 COLUMN | `COL`, `기둥`, `柱`, `边缘构件`/`邊緣構件`, `約束邊緣`/`约束边缘` |
| 보 BEAM | `BEAM`, `GIRDER`, `보`, `梁`, `桁`, `大梁`, `小梁` |
| 벽 WALL | `WALL`, `옹벽`, `벽`, `墙`/`牆`, `RC`, `SHEAR` |
| 축선 AXIS | `AXIS`, `AXN`, `GRID`, `CEN`, `축`, `통`, `軸`/`轴`, `通`, `网` |

### 1-2. 부정 키워드 (부재보다 **먼저** 매칭해 제외)
철근/치수/주석/해치/표제란/상세도 레이어는 도형이 아니라 **노이즈**다. `柱钢筋`·`柱尺寸`·`暗柱纵筋`처럼 부재 키워드를 포함한 파생 레이어를 먼저 걸러낸다.

```
钢筋/鋼筋/配筋/筋  尺寸  标注/標註  标高/標高  文字/文本  说明/說明
编号/編號  符号/符號  图号/圖號  图签/圖簽  图名/圖名  签名/簽名
填充  详图/詳圖  虚线/虛線  洞  表  철근  치수  문자  텍스트  해치
HATCH  TEXT  DIM  NUM  REIN  REBAR  DOTE  ELEV  MASONRY/조적/砖/砌(→조적벽 제외)
```

> **왜 먼저인가**: `1#2#结构图.dwg`에서 부정 키워드 없이는 기둥 **4366개**(철근 노이즈) → 적용 후 **435개**(실기둥). 자동필터가 부정 키워드 없이 영/한만 볼 땐 `砼柱`를 숨기고 철근 `WCOL_REIND`만 켜던 **치명 버그**가 있었다.

## 2. 부재 표현 방식 (기하)

### 2-1. 기둥
- **닫힌 폴리라인 사각형** = 표준. bbox + `minAreaRect`(회전 캘리퍼스)로 단면 b/h·회전각.
- ⚠️ **LINE 4개로 그린 기둥**(`COLU_BR` 등): 닫힌 도형이 아니라 개별 선 4개. → COLUMN 분류 레이어의 LINE을 **자동 클러스터링**(union-find)해 사각형 복원. (zg test.2: 기둥 18개·800×800)
- **타당성 필터**: 최대변>3000mm, 종횡비>6, 최소변<50mm면 제외(벽구간·철근 오인 차단).
- 기둥은 **닫힌 자기완결 도형이라 축척(scale)에 무관**하게 추출된다(벽/보와 대비).

### 2-2. 벽 / 보 (이중선 = 면쌍)
- 벽·보는 **두 평행선(양쪽 면)**으로 그려진다 → 상호최근접 면쌍 매칭 → 축선+두께/폭.
- ⚠️ **면쌍 매칭은 축척에 민감**: scale이 작으면(거대 미크롭 도면) 200mm 부재의 두 면이 sub-pixel(<1px)로 붙어 매칭 실패 → 한 층만 **CROP**하면 살아난다(scale 12×↑).
- **단일선 부재**: 면쌍을 못 찾으면 단일선으로 처리, 두께/폭은 그 도면 이중선 부재의 **중앙값**으로 추정(`widthEstimated: true`).

### 2-3. 边缘构件 (전단벽 단부기둥 / 暗柱)
- 剪力墙(전단벽)의 단부에 있는 concealed/boundary column. **기둥으로 취급**.
- ⚠️ 이 파일에선 상세표(暗柱表)가 아니라 **전층에 분산된 실부재**였음(국소밀도 측정으로 확인: 2.5m내 이웃 최대 2). 미크롭 시 1540개까지 뜨지만 크롭하면 184개로 정상화.

## 3. 문자 라벨 관례 (平法 flat method)

### 3-1. 集中标注 (집중표주) — 보 단면
형식: **`부호(경간수) 폭X춤`** = `KL(1) 200X400`
- **폭×춤(b×h)**: 평면 기하로는 **춤(depth)을 얻을 수 없다** → 라벨이 유일한 출처. 설계 명시값이라 기하 측정보다 정확.
- 정규식: `parseBeamLabel()` (geometry.ts). 철근·스터럽(`8@100/200(2)`, `2Φ14`) · 중문 주석은 정확히 배제.

| 부호 | 의미 |
|---|---|
| KL | 框架梁 (프레임보) |
| L | 次梁 (차보) |
| LL | 连梁 (연결보) |
| LLK | 连梁框 |
| KZL | 框支梁 |
| WKL | 屋面框架梁 (지붕 프레임보) |
| XL | 悬挑梁 (캔틸레버보) |

> **판별 원칙**: 레이어로 거르지 않고 **모든 TEXT를 규칙으로 판별**(벽부호가 보 레이어에 섞이는 등 관례가 뒤죽박죽). 주석 레이어가 자동필터로 숨겨져도 읽는다(그리드 라벨과 동일 방침). 부재 **분할 전**에 부여해 분할 조각들이 단면 속성을 상속.

## 4. 인코딩 (CJK 도면)

- 중국/대만 도면은 CJK 레이어명이 **코드페이지로 저장**됨 → 무조건 UTF-8 디코딩 시 깨짐(mojibake) → 분류 실패.
- ⚠️ **`$DWGCODEPAGE`는 무용**: libredwg가 소스값을 복사해 모든 파일에 `ANSI_936`이 박힘.
- **판별법 = B/H 비율**: UTF-8 무효문자(U+FFFD) 수 ÷ 고바이트(≥0x80) 수. 파일 크기·CJK 비율에 무관(dilution-independent).
  - UTF-8 = **0.00**, GBK = **0.55~0.69** → 임계 **0.15**
- 코드페이지→라벨: `ANSI_936`=gbk, `ANSI_950`=big5, `ANSI_949`=euc-kr, `ANSI_932`=shift_jis
- 구현: `src/utils/decodeDxf.ts` (워커·메인 폴백 공용). 三叶/12#(GBK) 복구, 1#2#(61MB UTF-8) 무손상 보존.

## 5. 도면 축척 (⭐ 중요 — 2× 관례)

> **출처**: 중국 구조 엔지니어 확인 (2026-07). 아래는 확정된 사실.

### 5-1. 규칙
중국 구조 시공도(특히 **PKPM/YJK 출도**)는 **기하를 실제 치수의 N배로 그리고, 치수 표기는 `DIMLFAC`(치수 선형축척)로 참값을 표시**하는 것이 **흔한 표준 기법**이다.
- `1#2#结构图.dwg` = **2배**로 그려짐 + `DIMLFAC=0.5`.
- 즉 **참값 = 기하 ÷ N**. 라벨 폭 200 → 기하 400, 축간 3300 → 기하 6600.
- 목적: 출도 시 선이 뭉개지지 않게 확대 + 국부 상세도.

### 5-2. ⚠️ 한 파일에 축척이 섞인다
한 DWG = 여러 **图框(도곽/시트)**, 각 도곽이 독립 축척. `1#2#`는 대부분 2× + 일부 평면 1:1.
→ **반드시 도곽(또는 CROP) 단위로 판정**. 전역 판정은 축척 혼재로 실패한다.

### 5-3. 판정법 = 치수문자 체인 (통계적 추론)
⚠️ **DIMLFAC를 직접 못 읽는다**: libredwg 변환 후 **DIMENSION 엔티티가 0개**(치수가 TEXT+선으로 분해됨). 네이티브 DXF(zg)도 동일. → 기하에서 추론하는 수밖에 없다.

**원리**: 치수문자는 재는 구간의 **중점**에 놓임 → 인접 두 치수문자 사이 실제거리 = 참값 기준 `(v1+v2)/2`.
```
k = 실제거리 ÷ ((v1+v2)/2) = 기하 ÷ 참값
```
- `1#2#`: 세로 체인 6쌍 연속 정확히 2.000, 전역 4916쌍 중 3072쌍 k=2.
- 구현: `src/utils/drawingScale.ts` (`analyzeDrawingScale`). 흔한 축척(1/2/2.5/4/5/10)으로만 스냅, 신뢰도는 체인 수가 아니라 **인접쌍 수로 가중**(잡음 체인 희석 방지).
- 보정: `DxfTransform.unitMm`(도면 1단위=몇 mm) → 월드 mm 환산. geometry의 mm 상수는 `mmPx = scale/unitMm` 경유.
- 검증: 라벨 대비 기하 측정 65개 중 **64개 오차 0**(보정 전 363개 전부 2배로 0개 일치).

### 5-4. 교훈
> **"도면 단위 = mm"라는 무의식적 단정이 오랜 오측정의 뿌리였다.** v0.36의 "벽 두께 440~600", v0.40의 "200/250"이 전부 **도면 단위**였고 참값은 그 절반. 새 도면을 볼 땐 **축척부터 의심**할 것.

## 6. DWG 버전 헤더

파일 첫 6바이트(latin1)로 판별. libredwg는 **AC1015(2000) 미만에서 불안정**.

| 코드 | 버전 | 코드 | 버전 |
|---|---|---|---|
| AC1006 | R10 (1988) | AC1018 | 2004 |
| AC1009 | R11/12 | AC1021 | 2007 |
| AC1012 | R13 | AC1024 | 2010 |
| AC1014 | R14 | AC1027 | 2013 |
| AC1015 | 2000 | AC1032 | 2018 |

> 로드 후 엔티티 0이면 버전을 읽어 명확히 안내(+직전 도면 보존). `AC1015` 미만(`code < 'AC1015'`)이면 구버전 경고.

## 7. 파일 구조 (대형 시트)

- 한 파일이 여러 평면/상세를 담아 **월드 범위가 수백 m**에 이른다(`1#2#`: x 121m).
- 범위>300m면 "한 층만 CROP" 배너 노출. 미크롭 + 벽15개↑ + 단일선>85%면 재추출 유도 힌트.
- **CROP 효과**: 선택 영역으로 fit scale 재계산 → scale 12×↑ → 이중선 면쌍·축선(sub-pixel이던 것)·축척 판정이 모두 살아남.
- ⚠️ **미래 개선(로드맵 #2)**: 图框 자동 분리 → 도곽별 축척 자동 판정(수동 CROP 제거). 제목텍스트(`结施-05 三层结构平面图`)로 층명 부여.

## 부록 A. 픽셀 바닥값 폭주 버그 계열 (반복 실수)

mm↔px 변환에서 **`Math.max(고정px, mm*scale)` 형태의 px 바닥값**은 작은 scale(거대 도면)에서 **월드 거리로 폭주**한다. 같은 버그가 두 번 나왔다:
- `gapCap`: 병합 갭 px 바닥값 30 → 작은 scale에서 100m로 폭주 → 보 메가병합 938m. **수정**: 월드 상한 `20000*scale`.
- `fMergePerp`: `Math.max(2, 30*scale)` = 2px = 421mm(크롭 scale) → 코드 자신의 불변식(`perpTol < 최소 부재두께`) 위반 → 200mm 부재 두 면 병합 → 두께 2배 오측정. **수정**: `Math.max(30*scale, 0.02)`.

> **원칙**: mm 상수를 px로 바꿀 땐 반드시 `mmPx`(=scale/unitMm) 경유. 순수 px 바닥값은 scale이 작을 때 의미가 뒤집힌다.
