# StruXureAI — 개발 로드맵

> 최종 목표: **벡터(DWG/DXF) → 정밀 구조부재 → 층기반 구조모델 → MIDAS Gen NX 핸드오프**
> ⚠️ **범위: 빠르고 정확한 "모델링 + 핸드오프"까지. 해석(하중/지진/풍/하중조합/해석실행)은 이 앱이 아니라 Gen NX에서 사용자가 함.**
> 시장: 대만/동남아(+한국) · 단계: 학습용(상용화 전) · 기준: **v0.27.1** (2026-06-24)

## 0. 지금까지 (완료) — "모델링 → 핸드오프 파이프라인"

```
DWG/DXF ─추출→ 구조부재 ─위상→ 절점그래프(FloorModel,mm) ─핸드오프→ MIDAS Gen NX / DXF
  ✅       ✅(벽 96%·기둥·보)   ✅(T자 노드공유)   ✅(단일 진실 소스)   ✅(API 라이브 실증)
```

- **추출**: 벽(축선+두께, 커버리지 96%), 기둥(단면+회전+gridRef, 실위치), 보(이중/단일선), 그리드+버블 라벨, 두께 양자화(대만/한국)
- **위상/데이터**: 끝점→기둥/교차점/T자 절점화 → **월드 mm 절점-부재 그래프(`FloorModel`)** 단일 진실 소스
- **핸드오프**: **MIDAS Gen NX API**(model 직접 소비, 절점 공유 직결, 단일층 **라이브 전송 성공**) · **DXF**(R12)
- **운영**: 다국어(한/영/번체), 도움말, 리사이즈 정합, Vercel 배포

> **핵심**: 모델링→핸드오프 파이프라인이 **동작**한다. 남은 본질은 **(1) 다른 실도면에도 정확히, (2) 다층(한 동), (3) 빠른 흐름.** 해석은 Gen NX 몫.

## 1. 단계별 로드맵 (목표까지)

각 단계는 독립적 가치를 가지며, 아래로 갈수록 최종 목표(구조해석)에 근접.
노력 S(작음)/M(보통)/L(큼) · 가치 ★(1~3).

### Stage A — Gen NX 핸드오프 완성 (대부분 완료)
> 모델을 Gen NX로 정확히 넘기는 길. (해석은 Gen NX 몫)

| # | 항목 | 상태 |
|---|---|---|
| A1 | 모델 연결성(절점 그래프, T자 접합 절점화) | ✅ v0.26~0.27 |
| A2 | MIDAS API 핸드오프(절점+BEAM/PLATE, model 직접 소비) | ✅ v0.19~0.27.1 (단일층 라이브 실증) |
| A3 | DXF 핸드오프(R12) | ✅ v0.22~0.23.1 |
| A4 | **다층(N≥2) 라이브 재전송 검증** | ⬜ Gen NX 재연결 후 |
| A5 | **DXF 임포트가 model 소비**(현재 px lines) | ⬜ 선택 |

### Stage B — 모델 규모/완성도 (모델링, 해석 아님)
| # | 항목 | 노력 | 가치 | 산출물 |
|---|---|---|---|---|
| B1 | **층별 다른 평면**(다중 DWG → BuildingModel) | L | ★★★ | 실제 다층 건물 모델 |
| B2 | **MIDAS Story Data(STRY) 등록**(층 정보) | S | ★★ | Gen NX에 층 표시 |
| B3 | 수직 연속성/표준층 관리(같은 gridRef 스택) | M | ★★ | 모델 무결성 |
| B4 | 보(Beam) 핸드오프 실데이터 검증 | S | ★★ | 보 있는 도면 필요 |

### Stage C — 추출 정밀도/일반성 (핵심 — "정확한 모델링")
| # | 항목 | 노력 | 가치 | 산출물 |
|---|---|---|---|---|
| C1 | **실도면 일반성**(대만/동남아 레이어 관례 다양화 대응) | M | ★★★ | B1F 외 도면에도 동작 |
| C2 | 남은 연결성(근접갭 자유단)·단일선 벽·비직교 코너·보 춤 라벨 | M | ★★ | 추출 품질↑ |
| C3 | DWG 렌더 보강(HATCH 등) | M | ★ | 배경 완성도 |

### Stage D — 워크플로 속도 ("빠르게")
| # | 항목 | 노력 | 가치 | 산출물 |
|---|---|---|---|---|
| D1 | **원클릭 흐름**(DWG 열기→자동필터→추출→Gen NX 전송 최소 클릭) | M | ★★ | 빠른 모델링 UX |
| D2 | 추출 옵션 프리셋·재시도·진행상태 | S | ★ | 사용성 |

### Stage E — 상용화 준비 / 보류
| # | 항목 | 비고 |
|---|---|---|
| E1 | 라이선스 정리(LibreDWG GPL, ArchCAD academic) | 상용 전 필수 |
| E2 | 성능·대용량·에러 핸들링 | |
| E3 | AI 인식(DWG 없는 사진/스캔, 대만 도면 파인튜닝) | 보류(초심의 빈 축) |

> ❌ **범위 외(해석)**: 하중·지진·풍·하중조합·해석실행·결과조회 — Gen NX에서 사용자가 수행. 이 앱은 모델만 넘긴다.

## 2. 추천 경로 (Milestones)

```
M1  C1 실도면 일반성        ← B1F 말고 다른 도면에도 정확히 (가장 실용적)
M2  B1 다층(다중 DWG) + B2 Story 등록 + A4 라이브 재전송  ← 실 다층 건물 핸드오프
M3  D1 원클릭 흐름          ← "빠르게"
M4  C2 남은 정밀도 / E 상용화
```

- **가장 실용적**: M1 — 지금은 B1F에 맞춰 검증됨. 다른 실도면에서도 되어야 "빠른 모델링 툴"이 됨.
- **규모**: M2 — 층별 도면을 받아 한 동 모델을 Gen NX로.
